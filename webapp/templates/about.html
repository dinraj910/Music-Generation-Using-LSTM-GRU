About Neural Music

Deep learning meets classical music composition

Project Overview

This project demonstrates the power of Recurrent Neural Networks (RNNs) for creative AI applications—specifically, generating musical melodies from learned patterns.

Using a dataset of classical music from Bach, Mozart, and Schubert (via the music21 corpus), three neural network architectures were trained to predict the next note in a musical sequence:

  • Stacked LSTM: Two-layer Long Short-Term Memory network
  • Stacked GRU: Two-layer Gated Recurrent Unit network
  • BiLSTM: Bidirectional LSTM (classifier demonstration)

How It Works

MIDI files are parsed using music21. Each note is converted to a pitch token (integers 0-87 for MIDI pitches 21-108, plus token 88 for REST).

Vocabulary: 89 tokens covering the full piano range (A0 to C8).

Using a sliding window approach with SEQ_LEN=32:

Input:  [note₁, note₂, ..., note₃₂]
Target: note₃₃

The model learns to predict the next note given the previous 32 notes.

Input(32) → Embedding(89, 64) 
    → LSTM/GRU(256, return_sequences=True)
    → Dropout(0.3)
    → LSTM/GRU(256)
    → Dropout(0.3)
    → Dense(89, softmax)

Output: Probability distribution over 89 possible next tokens.

1. Start with 32-note seed sequence
2. Model predicts next note probabilities
3. Sample using temperature scaling
4. Append prediction, slide window
5. Repeat for N notes

Temperature controls creativity: lower = more deterministic, higher = more random.

Why BiLSTM Can't Generate

Bidirectional LSTM reads sequences in both directions:

  • Forward pass: Left → Right (past context) ✅
  • Backward pass: Right → Left (future context) ❌

During training, all 32 notes exist, so both passes work. During generation, future notes don't exist yet—the backward pass has nothing to read.

BiLSTM is included to demonstrate this important concept. It may achieve better validation loss (by "cheating" with future info), but cannot compose new music.
